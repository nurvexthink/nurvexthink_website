\section{Overview}
Replace the current per-section static backdrops with a single \textbf{site-wide animated
ambient background}: a large, low-opacity \textbf{NT monogram watermark} that gently drifts and
``shines'', floating over slow-moving aurora glows and a faint blueprint grid. The effect is
\textbf{theme-aware} (distinct palettes for dark and light) and reacts subtly to the mouse for
depth. It renders \textbf{once}, behind all public pages, and is suppressed on \texttt{/admin}.

\subsection{Goals}
\begin{itemize}
  \item A premium, branded atmosphere that reinforces the NurvexThink mark on every public page.
  \item A tasteful ``shine'' that sweeps across the logo watermark, clipped to the letterforms.
  \item Smooth, GPU-friendly motion that never harms readability or Core Web Vitals.
  \item Fully theme-aware, honoring \texttt{prefers-reduced-motion}.
\end{itemize}

\subsection{Non-goals}
\begin{itemize}
  \item No canvas / WebGL / shader background (the hero already owns one GPU context).
  \item No per-page configuration or CMS control of the background.
  \item Not shown on admin pages (kept clean and functional).
\end{itemize}

% ===================================================================
\section{Locked decisions}
\begin{center}
\begin{tabularx}{\textwidth}{l Y}
\toprule
\textbf{Decision} & \textbf{Choice}\\
\midrule
Scope & Site-wide --- one fixed layer behind all public pages; excluded on \texttt{/admin*}\\
Style & Layered atmosphere --- centered NT monogram + drifting aurora + faint grid + shine\\
Motion & Ambient + mouse parallax (parallax disabled on touch and reduced-motion)\\
Intensity & Balanced --- clearly present, always behind content; watermark $\sim$6--10\% opacity\\
Technique & Pure CSS transforms + inline SVG (no canvas / WebGL)\\
\bottomrule
\end{tabularx}
\end{center}

% ===================================================================
\section{Approach}
Chosen: \textbf{CSS transforms + inline SVG}, animating only \texttt{transform} and
\texttt{opacity} so all motion runs on the compositor. Rejected alternatives: a WebGL / canvas
shader (a second always-on GPU context site-wide --- overkill, worse battery, conflicts with the
``lazy-load 3D so it never blocks paint'' rule) and animated SVG filters / \texttt{feTurbulence}
(CPU-heavy, hard to theme).

% ===================================================================
\section{Architecture}
One client component, mounted once in the root layout.

\begin{center}
\begin{tabularx}{\textwidth}{l Y}
\toprule
\textbf{Unit} & \textbf{Responsibility}\\
\midrule
\texttt{components/ambient-background.tsx} (\texttt{"use client"}) & The
\texttt{fixed inset-0 -z-10 pointer-events-none} layer, \texttt{aria-hidden}. Owns the
mouse-parallax effect and visibility rules. Returns \texttt{null} on \texttt{/admin*}.\\
\texttt{components/nt-mark.tsx} & The traced NT monogram as a reusable inline SVG, tintable via
\texttt{currentColor} / CSS vars. Swappable for an official SVG later.\\
\texttt{app/globals.css} & New keyframes (\texttt{aurora-drift}, \texttt{gleam-sweep},
\texttt{breathe}) and \texttt{.ambient-*} classes, theme-aware through existing tokens.\\
\texttt{app/layout.tsx} & Mounts \texttt{<AmbientBackground />} as the first child of
\texttt{<body>}.\\
\bottomrule
\end{tabularx}
\end{center}

\subsection{Layering (back to front)}
\begin{enumerate}
  \item \textbf{Base} --- existing \texttt{-{}-background} token (untouched; painted by \texttt{<body>}).
  \item \textbf{Aurora} --- 2--3 large soft radial-gradient blobs, slowly drifting + breathing.
  \item \textbf{Grid} --- the existing faint blueprint grid, full-page, edge-masked.
  \item \textbf{Monogram watermark} --- large, centered NT, low opacity, gently breathing.
  \item \textbf{Shine} --- a diagonal light band sweeping across the monogram every $\sim$6--8\,s,
        \textbf{clipped to the letter shapes} via an SVG \texttt{clipPath} so the gleam only rides
        the logo.
\end{enumerate}

The ambient layer lives at \texttt{-z-10} as a child of \texttt{<body>}. Per CSS paint order, a
negative \texttt{z-index} child paints \emph{above} the body's own \texttt{bg-background} fill but
\emph{below} all in-flow content --- so it is visible in transparent regions and correctly hidden
behind opaque \texttt{bg-card} surfaces.

\subsection{Mouse parallax}
A single \texttt{pointermove} listener (throttled with \texttt{requestAnimationFrame}) writes
normalized \texttt{-{}-mx} / \texttt{-{}-my} ($-1\ldots1$) CSS variables on the layer root. Each
layer translates a different magnitude toward the cursor (monogram least, aurora most) for depth.
The listener is \textbf{not attached} when \texttt{(hover: none)} (touch) or
\texttt{prefers-reduced-motion: reduce}.

\begin{tabularx}{\textwidth}{l Y Y}

\bottomrule
\end{tabularx}
\end{center}
Colors are driven by existing CSS custom properties, so the \texttt{.dark} class switch flips the
whole background automatically.

\end{itemize}

% ===================================================================
\section{Performance \& accessibility}
\begin{itemize}
  \item Animate \textbf{only} \texttt{transform} / \texttt{opacity}; \texttt{will-change} applied
        sparingly and scoped.
  \item \texttt{pointer-events: none} and \texttt{aria-hidden="true"} on the whole layer.
  \item Full \texttt{prefers-reduced-motion} support: the existing global guard in
        \texttt{globals.css} freezes animation; the parallax listener additionally self-disables.
  \item No layout thrash, no per-frame React state, no new dependencies.
\end{itemize}

\section{Asset note}

tinted or masked. We therefore \textbf{trace a clean transparent SVG} of the NT monogram for
\texttt{nt-mark.tsx}. It will closely match the mark; an official SVG can be dropped in later
without other changes.
